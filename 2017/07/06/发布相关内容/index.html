版本控制工具

本地式:在自己的存储版本库,只能单人开发使用
集中式: 把版本库集中放在服务器上
分布式:每人都有一个版本库,在本地进行开发,然后上传合并

git分支问题

一般在git创建之初的时候,有两个分支.
* master分支,主分支
* dev(develop分支,开发的分支)
* release分支

发布

大公司发布一般有指定的发布系统,通过发布系统进行发布
中小型公司一般人力往上发布,首先把项目进行打包,把服务器环境部署好,然后把文件放上去

  1. 服务器搭建的环境,和wamp非常像,但不是wamp,而是专门配置的
    • appach(或者IIS,windows系统中有自带的服务器软件
      或者Nginx(强大之处在于,它的反向代理配置简单) Node.js)
    • (若有后台语音)PHP(php环境),java(tomcat),ASP.NET
    • 数据路服务器,MySQL,oracle(甲骨文),MSSQL(微软),mongoDB
  2. 往服务器上传文件 FTP协议 file transfer protocal.
    服务器有IP地址，
    通过ftp客户端(flashFxp FileZila)

反向代理

解决跨域问题,前端要做的东西很少
但是,反向代理会增加公司服务器的压力,如果网站处于高并发(同时有很多人访问)的状态下,反向代理有可能导致服务器的崩溃,如果自己配置,一定要考虑服务器压力的情况.
